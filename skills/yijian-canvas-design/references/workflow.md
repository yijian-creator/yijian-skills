## Step 2: Design Philosophy

### 哲学文档结构

```markdown
# {哲学名称}

## 核心主张
（1段，点明美学世界观）

## 空间与形式
（如何通过空间和形状传达意义）

## 色彩与材质
（色彩系统、质感、降饱和度/高饱和度的选择）

## 尺度与节奏
（元素大小对比、重复节奏、视觉韵律）

## 构图与工艺
（构图原则、工艺标准——反复强调精心打磨、大师级执行）
```

### 哲学示例

**"数字经纬"**
制图学美学，流动与秩序并存。降饱和色彩，极淡网格，正交连接线，图例如制图框。每个节点都是地图上的坐标点，每条连线都是经纬线的延伸。

**"淡彩韵律"**
柔和色调中的信息流动。莫兰迪色系，留白呼吸，文字如水墨点缀。视觉重量轻盈，但信息密度不减。

**"Chromatic Silence"**
Color as the primary information system. Geometric precision where color zones create meaning. Typography minimal—small sans-serif labels letting chromatic fields communicate.

## Step 4: Canvas Creation

### ⛔ 生图前必须确认（前置检查）

在写任何渲染代码之前，必须完成以下三项确认：

1. **查字号层级表**：根据当前图片类型，查对应的字号体系
   - 文章配图（`yijian-article-illustrator` 生成）→ 查 `styles.md § 6.2`，字号范围 fs=6~14
   - 小红书观点卡（手动脚本）→ 查本文档附录"小红书图片规范"，字号范围 fs=13~52
   - **两套体系不能混用**

2. **确认画布尺寸**：查对应的实测字高表，验算空间是否充足
   - 文章配图 → 查 `styles.md § 6.5.1`（按 383/500/600px 分别列出）
   - 小红书图 → 查本文档附录"坐标系与字符宽度"（1080×1350px）

3. **换档规则**：字号和图不适配时，按以下优先级处理
   - 第一优先：调整布局（扩大容器/缩短文案），不动字号
   - 第二优先：在同一体系内换档（如 Body→Title），不跨体系
   - 第三优先：整体同比缩放（所有层级 ×1.2 或 ×0.8），不单独改某一个
   - **禁止**：单独把某一个元素的字号改到体系外的任意值

### 技术规范

**画布尺寸**（Python matplotlib）：
- 标准架构图：`figsize=(22, 16)`，`xlim=(0,22)`，`ylim=(0,16)`
- 海报/艺术品：`figsize=(16, 22)`（竖版）或 `figsize=(22, 16)`（横版）
- DPI：`200`（输出质量）

**⚠️ 顶部裁切防护（必须）**：`add_axes` 顶部必须留出物理空间，否则大字体上半部分会被裁掉：
```python
# ❌ 错误：坐标轴占满整个 figure，大字体顶部会被裁
ax = fig.add_axes([0, 0, 1, 1])

# ✅ 正确：顶部留 4% 物理空间（实测 fontsize=18 中文标题需要此空间）
ax = fig.add_axes([0, 0, 1, 0.96])
```
> 实测依据：900×600px，dpi=200，fontsize=18 中文标题，y=94 时顶部被裁；`add_axes([0,0,1,0.96])` 后上边距达到 2.7%，通过检测。

**字体规范**：
- 中文：`['PingFang SC', 'Heiti SC', 'STHeiti', 'Arial Unicode MS']`
- 英文/装饰：优先使用 `./canvas-fonts` 目录下的字体
- **画布尺寸不变，只调整 fontsize**——禁止同比放大画布和字体

### ⚠️ 字符宽度经验表（figsize=22×16，dpi=200 实测）

> 在写代码前，必须用此表计算文字实际占用宽度，确保节点/容器尺寸足够容纳文字。

| fontsize | 中文每字宽 | 中文每字高 | 英文每字符宽（估算） |
|----------|-----------|-----------|------------------|
| 28       | 0.502     | 0.471     | 0.19             |
| 22       | 0.394     | 0.370     | 0.15             |
| 20       | 0.358     | 0.344     | 0.13             |
| 16       | 0.287     | 0.273     | 0.11             |

**实测样本（可直接参考）：**

| 文本           | fontsize | 实测宽度 |
|----------------|----------|---------|
| `Node.js 代理` | 28       | 2.935   |
| `SSE 流式请求` | 22       | 2.279   |
| `server.js`    | 20       | 1.329   |
| `中文字符`（4字）| 28      | 2.006   |
| `中文字符`（4字）| 22      | 1.577   |

**计算公式：**
```
文本宽度 = Σ(中文字符数 × 每字宽) + Σ(英文字符数 × 每字符宽) + 空格数 × 每字符宽×0.5
节点最小宽度 = 文本宽度 + 左右内边距（建议各 0.35）
节点最小高度 = 字符高度 × 行数 + 上下内边距（建议各 0.30）
```

**示例计算（`Node.js 代理`，fontsize=28）：**
```
英文"Node.js " = 8字符 × 0.19 = 1.52
中文"代理" = 2字 × 0.502 = 1.004
总宽 ≈ 2.524，加左右padding 0.7 → 节点宽度至少 3.22
（实测 2.935，加 padding 建议节点宽 ≥ 3.5）
```

### ⚠️ 边界检查规则（⛔ 生成代码前必须逐项验证）

**1. 节点内文字不溢出**
```python
# 对每个节点，验证：
assert node_w >= text_width + 0.70   # 左右各 0.35 padding
assert node_h >= text_height * lines + 0.60  # 上下各 0.30 padding
# 有 subtitle 时，title 和 subtitle 各占一行，总高度 = 两行字高 + 行间距 0.15
```

**2. 节点之间不重叠**
```python
# 同行节点：相邻节点中心距 >= node_w + 0.5（最小间距 0.5 单位）
# 同列节点：相邻节点中心距 >= node_h + 0.5
assert abs(cx1 - cx2) >= node_w + 0.5  # 水平相邻
assert abs(cy1 - cy2) >= node_h + 0.5  # 垂直相邻
```

**3. 文字标签不与节点/箭头重叠**
```python
# 标签放在箭头旁侧时，x 偏移 >= 0.25 单位
# 标签的 y 范围不与任何节点的 y 范围相交
label_top = label_y + char_height / 2
label_bottom = label_y - char_height / 2
assert label_top < node_bottom or label_bottom > node_top
```

**4. 容器层底部留足箭头空间**
```python
# 层容器底部 到 下一个节点顶部 的间距 >= 1.5 单位
# 确保层间箭头 + 标签有足够空间显示
gap = container_bottom - next_node_top
assert gap >= 1.5
```

**5. 图例宽度适配内容**
```python
# 图例框宽度 = 色块宽(0.55) + 间距(0.25) + 最长标签文字宽 + 左右padding(0.4)
max_label_width = max(len(label) * char_width_per_fs for label in legend_labels)
legend_w = 0.55 + 0.25 + max_label_width + 0.4
```

**6. 所有元素在画布边界内**
```python
# 任何元素的边界不得超出 xlim/ylim
assert element_right <= xlim_max - 0.3   # 右边留 0.3 margin
assert element_left >= xlim_min + 0.3    # 左边留 0.3 margin
assert element_top <= ylim_max - 0.3
assert element_bottom >= ylim_min + 0.3
```

**每张图生成后必须用 `read_file` 读取图片进行视觉检查**，不能只看脚本无报错就结束。

### 回验清单（逐张执行）

**文字边界**：
- [ ] 每个文字块与相邻图块之间有明显间距，无视觉粘连
- [ ] 大标题顶部未被裁切（尤其 fontsize ≥ 28 的中文标题）
- [ ] 节点内文字未溢出节点边框

**图文关系**：
- [ ] 文字未与装饰图形（圆形、扇形、线条等）重叠
- [ ] 箭头/连线未穿过文字
- [ ] 图例与主体内容无重叠

**整体布局**：
- [ ] 左右两侧内容对称/对齐
- [ ] 底部说明文字未被裁切
- [ ] 水印可见，位置正确

### 发现问题的处理流程

1. **定位**：用实测脚本获取问题元素的精确宽高
2. **计算**：根据实测数据重新计算安全坐标
3. **修复**：用 `file_replace` 修改对应坐标，**修改前必须先 `read_file` 确认当前代码**
4. **重新生成并回验**：修复后重新生成图片，再次 `read_file` 确认
5. **循环上限**：同一张图修复不超过 3 轮，第 3 轮仍有问题则重新审视布局设计

## 附录：生图经验总结

### 图片命名规范

不同类型的图片必须在文件名前缀上体现类型，便于区分和管理：

| 前缀 | 类型 | 适用场景 |
|------|------|----------|
| `cover-` | 封面图 | 文章/内容封面，横版宽图 |
| `comparison-` | 对比图 | 两列对照、新旧对比、指标迁移 |
| `structure-` | 架构/层级图 | 系统架构、层级关系、组织结构 |
| `concept-` | 概念/哲学图 | 抽象概念对比、隐喻可视化 |
| `flow-` | 流程/步骤图 | 时序流程、步骤卡片、干预路径 |
| `matrix-` | 矩阵/全景图 | 多维分类、卡片网格、模式全景 |
| `timeline-` | 时间线图 | 历史演进、里程碑、版本迭代 |
| `infographic-` | 信息图 | 数据可视化、综合信息展示 |

**命名格式**：`{NN}-{type}-{slug}.png`
- `NN`：两位序号（01、02…）
- `type`：上表前缀（去掉连字符）
- `slug`：2-4个英文词，kebab-case

**示例**：`01-comparison-usability-metrics.png`、`03-concept-cognitive-model.png`

### 坐标系与字符宽度

**⚠️ 关键：不同画布尺寸的坐标系完全不同，字符宽高数值差异巨大，必须针对每种画布单独实测，不能跨坐标系复用数据。**

| 画布规格 | figsize | 坐标系 | 中文fs=22每字宽 | 中文fs=16每字宽 |

| 标准架构图 | (22,16) | xlim(0,22) | 0.394单位 | 0.287单位 |

| 文章配图方版 | (900px,500px) | xlim(0,100) | 27.2单位 | 14.8单位 |

**实测脚本**（换画布时必须重新跑）：

t = ax.text(0, 0, "测试文字", fontsize=22)
bb = t.get_window_extent(renderer=renderer)
inv = ax.transData.inverted()

p1 = inv.transform((bb.x1, bb.y1))
w, h = abs(p1[0]-p0[0]), abs(p1[1]-p0[1])

### 常见问题与修复方法

| 问题 | 原因 | 修复方法 |
|------|------|----------|
| 标题顶部被裁 | `add_axes([0,0,1,1])` 无顶部留白 | 改为 `add_axes([0,0,1,0.96])` |
| 文字与图块重叠 | 未实测字符宽高，靠估算放置 | 用实测脚本获取精确宽高后重新计算坐标 |
| 文字与文字重叠 | `va="center"` 时文字上下各延伸半个字高 | 间距 = 上方文字底部y + 安全间距(≥2单位) |
| 右侧内容溢出画布 | 文字宽度超出预期 | 实测宽度后调整起始x坐标或缩小fontsize |
| 边界检测报上边距0% | 大字体顶部超出ylim | `add_axes` 顶部留4%物理空间 |
| emoji/特殊符号显示为方块 | Hiragino Sans GB 不支持 emoji 字符 | 用文字或 ASCII 替代：`—`（破折号）、`→`（箭头）、`①②③`（带圆圈数字）均支持；禁止使用 ❌✅✓✗☑ 等 emoji |

### 图文分目录规范

同一篇文章/内容的图片，按用途分目录存放，**禁止混用**：

| 目录 | 用途 | 尺寸 | 风格 |
|------|------|------|------|
| `images/` | 文章配图（横版信息图） | 900×500 或 900×600 | 制图学/信息图风格，承载复杂信息 |
| `xhs-images/` | 小红书专属图（竖版观点卡） | 1080×1350（4:5） | 莫兰迪色系，大字标题，情绪感强 |

**小红书图片专项规范**：
- **尺寸**：1080×1350px，dpi=200，坐标系 xlim(0,100) ylim(0,100)
- **风格**：两种模式可选：
  - 「观点卡」：莫兰迪色系 + 大字标题 + 情绪感强，适合纯观点输出
  - 「图文一体」：白底极简 + 完整正文渲染进图，适合内容型文章（参考 momo科技谈 风格）
- **结构（观点卡）**：顶部色块标签（"观点 01"）+ 主标题（fs=36~52）+ 分隔线 + 内容区 + 水印
- **结构（图文一体）**：顶部页眉（账号标签 + 分隔线）+ 章节标签（红色竖条 + 标签文字）+ 主标题（fs=22~28）+ 分隔线 + 正文区 + 底部标签行 + 水印
- **一图一观点**：每张图只传达一个核心观点，不堆砌信息
- **字体兼容**：禁止使用 emoji（❌✅✓✗☑），Hiragino Sans GB 不支持；用 `—`、`→`、`①②③` 替代

**图文一体模式专项经验**（小红书渠道专用）：

1. **底部安全线**：必须设置 `FOOTER_Y = 14`，正文渲染循环中 `y < FOOTER_Y` 时自动停止，防止正文与底部标签重叠。底部标签固定在 y=6，分隔线固定在 y=12。
   ```python
   FOOTER_Y = 14
   def render_body(ax, body, y_start, line_gap=1.2):
       for text, fs, color, fw in body:
           if text == "":
               y -= 2.5; continue
           if y < FOOTER_Y:
               break   # 超出安全线自动截止
           ax.text(...)
           y -= char_h(fs) + line_gap
   ```

2. **字号体系（图文一体）**：
   - 主标题：fs=22~28（不用 fs=32，太大）
   - 正文：fs=14（不用 fs=16，内容多时放不下）
   - 强调句：fs=15
   - 页眉/标签：fs=12~13
   - 行间距：`line_gap=1.2`，空行减量：`y -= 2.5`（不用 3）

3. **内容分配原则**：单张图内容过多时，优先**拆分为两张**（如 02a 事实 + 02b 启示），而非压缩字号到 fs=12 以下。拆分点选在逻辑转折处（"事实"→"启示"，"现象"→"结论"）。

4. **文字对比检查（必须执行）**：生图后必须逐张做文字对比检查：
   - 列出脚本 body 里的所有文字行
   - 对照图片逐行确认是否全部显示
   - 发现截断 → 先减少空行间距，再考虑拆图
   - 发现重复 → 检查标题大字和正文是否表达了同一句话

5. **底图设计建议**：图文一体模式使用纯白底（`#FFFFFF`），不需要底图；观点卡模式可使用莫兰迪色系底图。

| 40       | 10.29     | 8.33      |
| 28       | 7.20      | 5.86      |

**边界自动检测脚本**（生成后必须运行）：
```python
from PIL import Image
import numpy as np

def check_margin(path, threshold=0.02):
    img = Image.open(path).convert('RGB')
    arr = np.array(img)
    h, w = arr.shape[:2]
    bg = np.mean([arr[0,0], arr[0,-1], arr[-1,0], arr[-1,-1]], axis=0)
    mask = np.abs(arr.astype(float) - bg).max(axis=2) > 15
    rows, cols = np.any(mask, axis=1), np.any(mask, axis=0)
    rmin, rmax = np.where(rows)[0][[0,-1]]
    cmin, cmax = np.where(cols)[0][[0,-1]]
    margins = {
        '上': rmin/h, '下': (h-1-rmax)/h,
        '左': cmin/w, '右': (w-1-cmax)/w
    }
    issues = [f"{k}边距不足({v*100:.1f}%)" for k,v in margins.items() if v < threshold]
    return margins, issues
```
