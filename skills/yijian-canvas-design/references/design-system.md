# 视觉设计系统

> 在创作任何视觉输出时，自动激活设计优化模式。设计思考过程在内部完成，用户只看到最终打磨好的成品。

---

## 设计质询清单

在交付任何视觉产出前，逐项自检：

### 字体设计
- [ ] 字体层级是否清晰且经过精心设计？（最多 3-4 个层级）
- [ ] 字号的递进是否有意义？（不是 14px 与 15px 的微小差别，而是像 12/16/24/36 这样有节奏的缩放）
- [ ] 行高是否与字体和行长相匹配？（正文 1.4-1.6 倍，标题则更紧凑）
- [ ] 字间距是增强了还是损害了可读性？（标题略微宽松，正文使用默认值）
- [ ] 使用不同字重是否能比单独使用字号创建更好的层级？
- [ ] 所选字体是否符合内容的调性？

### 色彩
- [ ] 色板是否服务于内容的目的？（蓝色代表信任，红色代表紧急，绿色代表增长）
- [ ] 是否精确使用了 2-4 种主色加上中性色？（而不是 7 种随机颜色）
- [ ] 对比度是否足以满足无障碍访问要求？（文本对比度至少 4.5:1）
- [ ] 颜色的使用是系统性的还是随意的？
- [ ] 降低饱和度能否提升精致感？
- [ ] 颜色是用于构建视觉层级还是仅仅为了装饰？

### 布局与间距
- [ ] 留白的使用是否是刻意为之？（而不是为了"填满页面"）
- [ ] 间距值是否遵循一定的比例？（例如 8/16/24/32/48/64，而不是 15/22/37 这样的随意数值）
- [ ] 网格系统是否创造了节奏感？（而不是随机定位）
- [ ] 相关元素之间的间距是否小于不相关元素？
- [ ] 是否有清晰的视觉入口和阅读路径？
- [ ] 不对称布局是否比居中布局更能引起兴趣？

### 视觉层级
- [ ] 能否在 1 秒内识别出最重要的元素？
- [ ] 是否恰好有 3 个视觉重要性层级？（而不是所有元素都在"大声尖叫"）
- [ ] 是通过尺寸/字重/颜色来创建层级，而不仅仅是位置吗？
- [ ] 次要元素是否被适当地弱化了？
- [ ] 移除某个元素能否提高清晰度？

### 数据可视化
- [ ] 图表类型是否最适合所要讲述的数据故事？（而不是对所有数据都使用默认的条形图）
- [ ] 坐标轴和标签是否清晰易读，毫不费力？
- [ ] 颜色是否用于编码信息，而不仅仅是装饰？
- [ ] 网格线是否足够微妙，能够融入背景？
- [ ] 展示更少的数据是否能讲述一个更清晰的故事？
- [ ] 比较是否在视觉上足够直观？

### 润色与工艺
- [ ] 边框/分隔线是必要的，还是仅仅是视觉噪音？
- [ ] 微妙的阴影或高差能否改善深度感知？
- [ ] 是否有一套一致的圆角系统？（例如 0, 4, 8, 16）
- [ ] 图标的风格和视觉重量是否一致？

## 技术目录

### 字体设计技术

**富有表现力的尺寸跳跃**
通过大胆的尺寸对比创造戏剧性（例如 72pt 的标题和 16pt 的正文）。
适用于：首屏区域、强调内容、杂志式布局。参考：Apple 产品页面、Bloomberg。

**紧凑行距以增强冲击力**
减小大标题的行高（0.9-1.1 倍）。
适用于：有力的声明、海报风格。参考：瑞士平面设计、马西莫·维涅里。

**宽松字距以营造奢华感**
增加大写字母的字间距（0.05-0.1em）。
适用于：高端品牌、章节标题、标签。参考：Dior、Net-a-Porter。

**字体色彩**
使用字重变化而非颜色来构建层级。
适用于：文本密集的界面、阅读体验。参考：Medium、Substack。

### 色彩技术

**单色深度**
使用单一色相的 6-8 种不同明暗度来构建整个色板。
适用于：精致的仪表盘、极简界面。参考：Stripe 仪表盘、Linear。

**强调色 + 中性色**
90% 的灰度色，仅搭配一种鲜艳的强调色。
适用于：吸引注意力、简洁的企业风格。参考：Tesla、Notion。

**深色模式的精致感**
使用纯黑背景搭配有层次感的界面元素（而不是 #222 这样的深灰色）。
适用于：营造高级感、现代应用程序。参考：Figma、GitHub 深色模式。

### 布局技术

**充足的留白**
使用 60-80% 的留白，而不是 20-30%。
适用于：奢侈品牌、强调焦点、创造呼吸感。参考：Muji、COS。

**不对称平衡**
元素权重不均等，但保持视觉上的平衡感。
适用于：落地页、叙事性布局。参考：Apple 活动页面、Airbnb。

**重叠图层**
通过在不同 Z 轴上重叠元素来创造深度。
适用于：增加视觉趣味、营造空间深度。参考：Stripe 主页、现代 SaaS 网站。

**破格布局**
建立网格，然后为了强调而刻意打破它。
适用于：作品集、杂志式布局。参考：Awwwards 获奖作品、Pentagram。

### 数据可视化技术

**通过弱化进行强调**
将除焦点之外的所有数据都进行去色处理。
适用于：高亮特定指标、进行对比。参考：《用数据讲故事》、纽约时报图表。

**注解优先图表**
先呈现洞见文字，图表作为叙事的支撑。
适用于：报告、演示文稿、解释性可视化。参考：《经济学人》、数据新闻。

**小型多图**
对不同细分领域重复使用同一种图表类型。
适用于：比较分析、跨类别趋势。参考：爱德华·塔夫特、FiveThirtyEight。

### 润色技术

**微妙渐变**
5-10% 的亮度差异，而不是 50%。
适用于：增加深度、精致感、背景。参考：Apple UI、iOS 设计语言。

**一致的圆角系统**
最多使用 2-3 个圆角值（例如 4px, 8px, 16px）。
适用于：按钮、卡片、容器。参考：Tailwind 默认值、现代设计系统。

## 设计提升流程（7 阶段）

### 阶段 1：基础审核
- 检查功能完整性（它是否能正常工作？）
- 识别视觉层级问题（什么是最重要的？）
- 标记随意的决策（随机的颜色、间距、尺寸）

### 阶段 2：字体设计提升
- 选择有明确意图的字体
- 建立字体缩放比例（非线性：12/16/24/36/48/72）
- 系统地设置行高（正文 1.6 倍，标题 1.2 倍，大标题 1.0 倍）
- 优先使用字重变化，其次是颜色变化
- 检查可读性（行长 45-75 个字符）

### 阶段 3：色彩优化
- 将色板减少到 2-3 种颜色 + 中性色
- 确保系统化使用（而非装饰性）
- 检查对比度（至少满足 WCAG AA 标准）
- 尽可能降低饱和度，优先考虑单色方案

### 阶段 4：空间系统
- 定义间距比例（8/16/24/32/48/64/96）
- 应用一致的内部填充（padding）
- 将相关元素组合得比不相关元素更近
- 将留白增加 30-50%

### 阶段 5：视觉层级
- 确保有一个清晰的焦点
- 精确创建 3 个重要性层级
- 弱化次要内容（而不仅仅是放大主要内容）
- 移除相互竞争的视觉元素

### 阶段 6：润色环节
- 对元素进行视觉对齐（而不仅是数学对齐）
- 如果深度有助于理解，则添加微妙的阴影
- 确保一致的圆角
- 检查图标的一致性和视觉重量

### 阶段 7：精简测试
- 移除一个元素——清晰度是否提升了？
- 减少一种颜色——它是否仍然有效？
- 增加留白——是否感觉更高级？
- 移除边框/分隔线——它是否必要？
- 简化图表——故事是否依然清晰？

## 品味过滤器

### ❌ 避免
- 彩虹色方案（系统地使用 2-3 种颜色）
- 正文居中对齐（为了可读性请左对齐）
- 装饰性渐变（用于营造深度，而非装饰）
- 对所有元素使用投影（层级感应是刻意为之）
- 为了"设计感"而设计（没有目的的装饰）
- 过度居中布局、紫色渐变、统一大圆角、Inter 字体（AI Slop 四件套）

### ✅ 拥抱
- 充足的留白（对自己展示的内容充满信心）
- 系统性约束（有限的色板、字体比例、间距）
- 有意为之的不对称（比居中更有趣）
- 微妙的精致感（优化胜于浮华）
- 清晰的层级（一个声音响亮，其他声音作为支撑）
- 将无障碍性作为设计约束（而非事后弥补）
