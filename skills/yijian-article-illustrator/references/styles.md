## ⛔ 全局硬约束：中文输出

无论目标渠道是哪一个，所有产出**必须为简体中文**：

- 改写后的正文、所有标题、所有小标题
- 图片的 `title`、`prompt`、`key_elements`、`philosophy_hint`

## 六、配图渲染规范（Matplotlib 实战经验）

> 以下规范来自多轮实际渲染调试，必须严格遵守，否则大概率出现溢出、重叠、偏移等问题。

### 6.1 垂直居中：二分法像素校准（最佳实践）

**关键认知：坐标中心 ≠ 像素中心。** 大字号元素（如 fs=14 的标题）在像素层面的"视觉重量"远大于小字号元素。纯坐标计算的"居中"在人眼看来往往偏上。

#### 6.1.1 为什么公式精算不够

```
坐标系里，标题 y=77，底部文字 y=29
坐标中心 = (77+29)/2 = 53 → 看起来接近画布中心50 → "应该居中"
但实际像素：标题渲染后顶部在 y=32px，底部文字底边在 y=311px
像素中心 = (32+311)/2 = 172px → 画布中心250px → 偏上78px！
```

**原因**：fs=14 标题渲染后占 ~50px 高度，fs=6.5 小文字只占 ~20px。坐标系里它们都是"一个点"，但像素占比差 2.5 倍。

#### 6.1.2 正确做法：三步走

**Step 1: 先给内容足够空间**

卡片/色块等容器的高度必须**先保证内容不拥挤**：
- 容器内有 3 层内容（编号+标题+描述）→ 容器高度 ≥ 30 坐标单位
- 容器内有 2 层内容 → 容器高度 ≥ 20 坐标单位
- 用 § 6.5 空间预算公式验算

> ⚠️ **绝对禁止为了"居中"而压缩容器高度**。先保证内容不挤，再调居中。

**Step 2: 用初始值渲染一版**

先用经验初始值 `y_top = 50 + estimated_H/2` 渲染出图片。

**Step 3: 二分法像素校准**

用 PIL 像素扫描检测实际上下边距，二分法逼近最佳 `y_top`：

```python
from PIL import Image
import numpy as np

def measure_margins(image_path, bg_rgb=(250, 248, 244), threshold=10):
    """像素级测量上下边距"""
    img = Image.open(image_path)
    arr = np.array(img)
    h, w = arr.shape[:2]
    bg = np.array(bg_rgb)
    first = last = 0
    for row in range(h):
        diff = np.abs(arr[row, :, :3].astype(int) - bg)
        if np.any(diff.max(axis=1) > threshold):
            first = row; break
    for row in range(h-1, -1, -1):
        diff = np.abs(arr[row, :, :3].astype(int) - bg)
        if np.any(diff.max(axis=1) > threshold):
            last = row; break
    return first, h - 1 - last  # top_margin, bottom_margin

# 二分法找最佳 y_top
lo, hi = 70.0, 90.0
for _ in range(15):
    mid = (lo + hi) / 2
    render_with_ytop(mid)  # 用 mid 作为 y_top 渲染图片
    top_m, bot_m = measure_margins("output.png")
    if abs(top_m - bot_m) <= 5:
        break
    if top_m > bot_m:   # 内容偏下
        lo = mid        # y_top 要增大（上移）
    else:               # 内容偏上
        hi = mid        # y_top 要减小（下移）
# 最终 mid 就是最佳 y_top
```

**验收标准**：`|top_margin - bottom_margin| ≤ 10px`，比值在 0.8~1.2 之间。

主标题 (fs=14) 与英文副标题 (fs=7) 的 y 间距必须 ≥ 7 坐标单位（中心到中心）。

```python
# ❌ 错误：间距 4.5，标题和副标题视觉重叠
```

### 6.4 常见 Bug 及修复

| Bug | 原因 | 修复 |
|-----|------|------|
| 标签卡片第 N 个无背景框 | y 坐标超出色块范围 | 增大色块高度，或减小标签间距；间距公式：`间距 = 字号 + 2` |
| 左右两列文字重叠 | 卡片宽度不足放两列 | 改为单行居中展示，合并 label+value |
| 底部文字被截断 | 单行文字超出画布宽度 | 拆成两行，每行独立 `ax.text`，y 坐标各差 4 单位 |
| Unicode 箭头乱码 | matplotlib 字体不支持 `→` 等符号 | 改用 `FancyArrowPatch`，禁止在文字中使用 `→ ← ↑ ↓` |
| 内容整体偏上 | 标题固定在顶部，未按内容高度居中 | 见 6.1 垂直居中方案 |
| 描述与标签区域重叠 | 描述文字和标签卡片共用同一 x 起点 | 描述放顶部，标签放底部，y 坐标错开 |
| 标题与副标题重叠 | 主标题(14)实测高度 6-10 单位，间距太小 | 标题与副标题中心点距离 ≥ 主标题实测高度 + 2 |
| 底色框与相邻元素重叠 | 只检查了文字边界，忽略了 `bbox` 底框 | 带底色框的元素，用**框的边界**参与碰撞检测（见 § 6.4.1） |
| 内容偏上/底部空白大 | 坐标中心≠像素中心，大标题视觉权重偏上 | 二分法像素校准（见 § 6.1.2 Step 3） |
| 为居中压缩卡片高度 | 卡片内容挤成一团，编号/标题/描述粘连 | **先保证内容空间，再调居中**。禁止为居中牺牲可读性 |
| file_replace 导致代码丢失 | 替换范围过大，误删中间代码 | 替换前必须 read_file 确认上下文，替换范围尽可能精确 |

#### 6.4.1 底色框整体碰撞检测（重要！）

当文字带有底色框（`bbox=dict(boxstyle="round,pad=0.3", ...)`）时，**框的边界 > 文字边界**。碰撞检测必须用框的边界，而非文字边界。

```python
# ── 错误做法 ──
# 只检查文字 "01" 的高度（约 4.8 坐标单位）
# 实际底色框因为 pad=0.3，高度 ≈ 文字高度 + 2*pad*fontsize ≈ 10+

# ── 正确做法 ──
# 带 bbox 的元素，碰撞边界 = 文字边界 + pad 扩展
def get_bbox_with_pad(text_y, text_h, pad, fontsize):
    """计算带底色框的元素的实际边界"""
    pad_coord = pad * fontsize * 72 / canvas_height_px  # pad 转坐标单位（近似）
    return (text_y - text_h/2 - pad_coord, text_y + text_h/2 + pad_coord)
```

**规则**：
1. 文字与底色框视为**一个整体**
2. 相邻元素的净间距必须以**框边界**为基准计算
3. 底色框的 `pad` 值越大，占用空间越大，必须计入空间预算

#### 6.5.2 净间距公式

两个文字元素之间的**净间距**（边缘到边缘，不是中心到中心）：

```python
net_gap = abs(y_center_A - y_center_B) - text_height_A / 2 - text_height_B / 2
# net_gap 必须 ≥ 2（坐标单位），否则视觉上会粘连/重叠
```

#### 6.5.3 容器空间预算

**渲染前必须验算**：容器内所有元素的实际占高之和 + 间距之和 ≤ 容器高度。

```python
# 实测数据查找表（dpi=200，中文字体）
TEXT_HEIGHT = {
    # (fontsize, canvas_height_px) → 实测占高
    (6, 383): 4.01, (6, 500): 3.07, (6, 600): 2.56,
    (7, 383): 4.96, (7, 500): 3.80, (7, 600): 3.17,
    (7.5, 500): 3.8, (7.5, 600): 3.4,
    (8, 383): 6.01, (8, 500): 4.60, (8, 600): 3.83,
    (9, 383): 6.27, (9, 500): 4.80, (9, 600): 4.00,
    (11, 500): 5.80, (11, 600): 4.83,
    (14, 383): 9.92, (14, 500): 7.60, (14, 600): 6.33,
}

def get_text_height(fontsize, canvas_height_px):
    """查表获取实测高度，未命中则线性插值"""
    key = (fontsize, canvas_height_px)
    if key in TEXT_HEIGHT:
        return TEXT_HEIGHT[key]
    # fallback: 用最近的 canvas 尺寸线性插值
    return fontsize * 0.54  # 粗估

def check_space_budget(container_height, elements, canvas_height_px):
    """
    container_height: 容器在坐标系中的高度
    elements: list of (fontsize, count)  # 该字号的文字行数
    """
    total = 0
    for fontsize, count in elements:
        h = get_text_height(fontsize, canvas_height_px)
        total += h * count
    total_lines = sum(count for _, count in elements)
    total += (total_lines - 1) * 2  # 净间距最小值 2
    if total > container_height:
        raise ValueError(f"空间不足：需要 {total:.1f}，容器仅 {container_height}")
```

**示例**（02-comparison 每个卡片，canvas=500px，新字号）：
- 卡片标题（字号 9）× 2 行（换行）= 4.80 × 2 = 9.60
- 条目（字号 7.5）× 4 行 = 3.80 × 4 = 15.20
- 间距 5 × 2 = 10
- **合计 = 34.80**，卡片高度 52 → ✅ 充裕

```python
# 标准三列布局
```

### 6.7 四层自查清单（每张图渲染后必查）

> 每次生成设计内容后，必须严格按照以下顺序进行自查，每一项必须检查，不得跳过，全部通过后才能输出最终结果。

#### 【第一层：布局检查】

1. **全局位置**：整体内容是否在画布中正确对齐（水平/垂直）。**垂直居中必须用像素级验证**（见 § 6.1.2 Step 3），不能只看坐标
2. **局部位置**：每个组件内部元素是否对齐
3. **元素覆盖**：所有元素之间是否存在相互遮挡/覆盖
   - 3.1 装饰性元素（箭头、标注、图标）是否与主体内容重叠
   - 3.2 标注文字是否在被标注对象的合理位置（外部/附近，不是内部）
   - 3.4 **带底色框的元素**（`bbox` 参数），碰撞检测必须用**框边界**而非文字边界（见 § 6.4.1）
   - 3.3 所有元素的边界框是否两两不相交
   
   **执行规范（必须严格遵守）**：
   ```
   ① 列出所有装饰性元素的坐标范围（x_min, x_max, y_min, y_max）
   ② 列出所有主体元素的坐标范围
   ③ 两两比较：装饰性元素坐标是否在主体元素坐标范围内
   ④ 如果在范围内，标注位置是否合理（应该在外部/附近，不是内部）
   ⑤ 不允许只凭眼睛"扫一眼"就说没问题，必须量化检查
   ```
   
4. **边界溢出**：是否有元素超出画布/容器边界
5. **间距统一**：元素**净间距**是否 ≥ 2 坐标单位（见 § 6.5.2 净间距公式，不是中心点距离）
6. **层级关系**：前后层级关系是否正确，是否有误压情况

#### 【图片查看规范】

查看渲染结果图片时必须：
- 逐区域检查（左上/右上/左下/右下/中心），不能只"扫一眼"
- 对每个装饰性元素，确认其坐标是否在主体外部
- 发现可疑重叠时，必须回到代码检查坐标计算逻辑

#### 【第二层：文字检查】

7. **字体规范**：所有文字字体是否符合设计规范
8. **字号层级**：标题/正文/辅助文字字号层级是否正确

   **检核方法（必须逐项执行，不能只凭眼睛判断）**：
   
   ① **对照 § 6.2 字号层级表**，列出图中每个文字元素的实际 fontsize，逐一确认是否在体系内：
   ```
   图表主标题 → 必须是 14
   副标题（英文）→ 必须是 7
   卡片/模块标题 → 必须是 9
   正文/条目 → 必须是 7 或 7.5
   标签/小注 → 必须是 6.5
   最小文字 → 必须是 6
   ```
   
   ② **检查是否混用了两套字号体系**：
   - 文章配图（本 skill 生成）只能用 fs=6~14 的体系
   - 小红书观点卡（手动脚本）只能用 fs=13~52 的体系
   - 同一张图里出现跨体系的字号（如 fs=9 和 fs=22 混用）→ ❌
   
   ③ **检查是否有字号跑到体系外的任意值**（如 fs=11、fs=13、fs=15）：
   - 体系外的字号 → ❌，必须换档到最近的体系内值
   
   ④ **同类元素字号必须统一**：
   - 所有卡片标题字号相同 → ✅
   - 不同卡片标题字号不同 → ❌
9. **文字截断**：是否有文字显示不全/被截断
10. **文字覆盖**：文字是否压在其他元素上导致不可读
11. **对比度**：文字与背景颜色对比度是否足够清晰
12. **行高字距**：行高/字间距是否符合规范（建议 linespacing=1.5）

#### 【第三层：视觉检查】

13. **颜色规范**：所有颜色是否在规定色板范围内
14. **圆角统一**：圆角大小是否统一符合规范
15. **图标尺寸**：图标大小是否统一规范
16. **阴影效果**：阴影参数是否符合规范
17. **透明叠加**：透明度/叠加混合效果是否正确

#### 【第四层：完整性检查】

18. **内容为空**：是否考虑内容为空时的展示状态
19. **文字过长**：是否考虑文字内容过长时的处理方式
20. **状态完整**：需要的交互状态是否完整（正常/禁用/错误等）
