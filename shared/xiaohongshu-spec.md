# 小红书「图文一体」模式技术规范

> **本文档为共享规范**，被 `yijian-canvas-design` 和 `yijian-article-illustrator` 共同引用。
> 
> **适用场景**：深度分析类文章、知识科普、商业逻辑拆解等需要完整阅读的内容型笔记。
> 
> **与"观点卡"模式的区别**：图片本身就是内容载体，正文文字直接渲染在图里，md 正文仅保留标题+钩子+标签作为精华版。

---

## 1. 技术规范

| 项目 | 规格 |
|------|------|
| **画布尺寸** | 1080×1350px，dpi=200，坐标系 xlim(0,100) ylim(0,100) |
| **背景图** | 用户提供（如 `/path/to/bg.jpeg`），原始尺寸需缩放到 1080×1350 |
| **蒙版** | 白色半透明矩形 `Rectangle((0,0), 100, 82, alpha=0.93)`，只覆盖内容区 y=0~82，避开顶部账号区 y=85~100 |
| **封面** | 不加背景，纯白底 + 暖沙色顶栏 |
| **其他图** | 全部叠加背景图 + 蒙版 |

### 1.1 安全区域（坐标系 0~100）

- **顶部账号区**：y=85~100（背景自带，内容严禁覆盖）
- **内容安全区**：y=14~82（正文渲染范围）
- **底部标签区**：y=0~14（FOOTER_Y=14，标签固定在 y=6，分隔线在 y=12）
- **中间水印**：背景图可能包含淡灰色装饰字（如"壹""間"），位于 y=30~70，蒙版会将其淡化

---

## 2. 字号体系

| 层级 | 字号 | 用途 |
|------|------|------|
| 主标题 | fs=22~28 | 每张图的大标题（不用 fs=32，太大占空间） |
| 正文 | fs=13 | 卡片内的说明文字（适配压缩后的安全区） |
| 强调句 | fs=14 | 关键论点、数据突出 |
| 页眉/章节标签 | fs=12~13 | 顶部账号标签、章节标识 |
| 行间距 | `line_gap=0.9` | 正文行间距 |
| 空行减量 | `y -= 2.0` | 空行时的 y 偏移量 |

## 4. 结构模板

```
┌─────────────────────┐
│ [背景账号区 y=85~100] │  ← 背景自带，不渲染
├─────────────────────┤
│ [暖沙竖条] 章节标签    │  ← y=82 开始
│                     │
│ 主标题（fs=22~28）     │
│ ─────────────────── │  ← 分隔线
│                     │
│ 正文区（fs=13）        │  ← y=82~14
│ · 要点1             │
│ · 要点2             │
│                     │
├─────────────────────┤
│ 底部标签（y=6）        │  ← FOOTER_Y=14 以下
└─────────────────────┘
```

---

## 5. 内容分配原则

1. **单张图内容过多时优先拆图**：在逻辑转折处拆分（如"事实"→"启示"，"现象"→"结论"），而非压缩字号到 fs=12 以下
2. **底部安全线强制截止**：渲染循环中 `if y < FOOTER_Y: break`，防止正文与底部标签重叠
3. **文字对比检查（必须执行）**：生图后逐张核对脚本 body 与图片实际显示，发现截断先减少空行间距，再考虑拆图；发现重复则检查标题大字和正文是否表达同一句话

---

## 6. 水印处理

- 右下角署名水印（如"钧弈"）**必须移除**，避免与背景图账号区冲突
- 调用 `draw_watermark()` 时直接 `pass` 或注释掉

## 7. 示例代码片段

```python
def new_canvas(with_bg=True):
    fig = plt.figure(figsize=(1080/200, 1350/200), dpi=200)
    ax = fig.add_axes([0, 0, 1, 0.98])
    ax.set_xlim(0, 100); ax.set_ylim(0, 100)
    ax.set_facecolor(WHITE); fig.patch.set_facecolor(WHITE)
    ax.set_xticks([]); ax.set_yticks([])
    for s in ax.spines.values(): s.set_visible(False)

    if with_bg:
        bg = mpimg.imread(BG_PATH)
        ax.imshow(bg, extent=[0, 100, 0, 100], aspect='auto', zorder=0)
        # 白色蒙版只覆盖内容区，避开顶部账号区
        overlay = Rectangle((0, 0), 100, 82, facecolor='white',
                            alpha=0.93, edgecolor='none', zorder=1)
        ax.add_patch(overlay)
    return fig, ax

def render_body(ax, body, y_start, line_gap=0.9):
    y = y_start
    for text, fs, color, fw in body:
        if text == "":
            y -= 2.0; continue
        if y < FOOTER_Y:  # 底部安全线强制截止
            break
        ax.text(MARGIN_X, y, text, fontsize=fs, color=color,
                va="top", ha="left", fontweight=fw, zorder=2)
        y -= char_h(fs) + line_gap
    return y
```

## 8. 重要提醒

1. 此模式与 workflow.md 中定义的"观点卡"模式（莫兰迪色系+大字标题+情绪感强）是**两种独立的小红书风格**，根据用户需求选择
2. 背景图必须由用户提供，skill 不负责生成背景
3. 蒙版 alpha=0.93 是经过实测的最佳值，既能淡化中间水印，又不会让背景完全消失
4. 如果用户未提供背景图但要求"图文一体"，默认使用纯白底（`with_bg=False`），退化为无背景版本
